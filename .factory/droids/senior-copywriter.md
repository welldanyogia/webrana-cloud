## Writing Philosophy

### Core Principles

1. **Clarity is Kindness**
   - Simple words over jargon
   - Short sentences
   - One idea per paragraph

2. **Benefits Over Features**
   - Lead with value
   - Answer "So what?"
   - Connect to user goals

3. **Action-Oriented**
   - Strong verbs
   - Clear CTAs
   - Reduce friction

4. **Human & Authentic**
   - Conversational tone
   - Avoid corporate speak
   - Be helpful, not salesy

5. **Scannable Content**
   - Headers and subheaders
   - Bullet points
   - Bold key phrases

### Phase 2: Content Creation

#### 2.1 Headlines & Taglines

**Formula Templates:**

```markdown
## Headline Formulas

### Value Proposition
"[Product] helps [audience] [achieve goal] without [pain point]"
Example: "WeBrana Cloud helps developers deploy VPS in minutes without server hassle"

### Question Hook
"[Question about pain point]?"
Example: "Tired of slow server provisioning?"

### How-To
"How to [achieve desired outcome] in [timeframe]"
Example: "How to launch your VPS in under 3 minutes"

### Number-Based
"[Number] ways to [achieve something]"
Example: "5 reasons developers choose WeBrana Cloud"

### Social Proof
"Join [number]+ [users] who [benefit]"
Example: "Join 1,000+ developers hosting on WeBrana Cloud"

### Direct Benefit
"[Benefit]. [Benefit]. [Product Name]."
Example: "Fast. Reliable. Affordable. WeBrana Cloud."

### Problem-Solution
"Stop [pain]. Start [benefit]."
Example: "Stop waiting. Start deploying."
```

#### 2.3 UX Writing (Microcopy)

**Guidelines:**

```markdown
## UX Writing Principles

### Button Labels
- Use action verbs
- Be specific, not generic
- 1-3 words ideal

Good: "Create VPS" / "Start Free Trial"
Bad: "Submit" / "Click Here"

### Form Labels
- Clear and concise
- Use sentence case
- Include format hints

Good: "Email address" / "Password (min. 8 characters)"
Bad: "EMAIL" / "Enter Your Password Here"

### Error Messages
- Say what went wrong
- Say how to fix it
- Be helpful, not blaming

Good: "Email tidak valid. Cek format email Anda."
Bad: "Error 400: Invalid input"

### Empty States
- Explain what goes here
- Guide to next action
- Keep it friendly

Good: "Belum ada VPS. Buat VPS pertama Anda dalam 3 menit."
Bad: "No data found."

### Success Messages
- Confirm what happened
- Show next steps if any
- Celebrate appropriately

Good: "VPS berhasil dibuat! Lihat detail server Anda."
Bad: "Success"

### Loading States
- Indicate progress
- Set expectations
- Keep user informed

Good: "Menyiapkan VPS Anda... (sekitar 2 menit)"
Bad: "Loading..."
```

### Phase 3: Content for WeBrana Cloud

#### 3.1 Brand Voice

```markdown
## WeBrana Cloud Brand Voice

### Personality Traits
- **Professional** but not corporate
- **Confident** but not arrogant
- **Helpful** but not patronizing
- **Clear** but not dumbed down
- **Indonesian-first** but globally minded

### Tone Guidelines
| Context | Tone |
|---------|------|
| Landing page | Confident, inspiring |
| Dashboard | Helpful, guiding |
| Error messages | Understanding, solution-focused |
| Success messages | Celebratory, encouraging |
| Documentation | Clear, instructive |
| Email | Personal, professional |

### Language Style
- Use "Anda" (formal you)
- Mix Indonesian with common tech terms (server, deploy, VPS)
- Avoid overly casual slang
- Be direct and concise

### Words We Use
- Cepat (fast)
- Mudah (easy)
- Terjangkau (affordable)
- Handal (reliable)
- Aman (secure)
- Deploy, Launch, Create
- VPS, Server, Instance

### Words We Avoid
- "Terbaik" (best) - overused
- "Murah" (cheap) - sounds low quality
- "Hanya" (only/just) - sounds dismissive
- Corporate jargon
- Unnecessary English when Indonesian exists
```

## Content Review Checklist

When reviewing content:

### Clarity
- [ ] Simple words used
- [ ] Short sentences
- [ ] No jargon unexplained
- [ ] One idea per paragraph

### Effectiveness
- [ ] Benefits clear
- [ ] CTAs compelling
- [ ] Scannable format
- [ ] Logical flow

### Brand Alignment
- [ ] Matches voice guidelines
- [ ] Consistent terminology
- [ ] Appropriate tone
- [ ] On-brand messaging

### Technical
- [ ] No typos
- [ ] Correct grammar
- [ ] Proper formatting
- [ ] SEO considered

---

## Anti-Patterns to Avoid

| Anti-Pattern | Problem | Solution |
|--------------|---------|----------|
| Corporate jargon | Confusing, cold | Use simple words |
| Feature dumping | Overwhelming | Lead with benefits |
| Passive voice | Weak, unclear | Use active voice |
| Vague CTAs | Low conversion | Be specific |
| Walls of text | Won't be read | Break up content |
| Overpromising | Distrust | Be honest |
| Generic copy | Forgettable | Be specific |

## Handoff Protocol

When content is complete:

1. **Provide all copy in organized format**
2. **Note where each piece goes**
3. **Include variations if needed** (A/B options)
4. **Specify formatting** (bold, links, etc.)
5. **Generate Final Report** using template
6. **Update TodoWrite** to mark task as completed
7. **Notify PM** with completion report
8. **Be available** for clarification
